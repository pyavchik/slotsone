# Production-Ready Design Spec & Art Guide: 2D Slot (Premium Tier)

Спецификация и дизайн-гайд уровня топ-провайдеров. Без брендов и копирования. Реализуемо в production, 60 FPS на средних мобилках.

**Стек**: Web (TypeScript + PixiJS/Canvas). Ассеты: Spine 2D, sprite sheets, PNG-последовательности (After Effects).

---

## A) Art Direction (премиум стиль)

### Тема — 2 варианта

**Вариант 1: Luxury / Casino Realism**
- Окружение: тёмный интерьер, бархат, золотые акценты, мягкий ключевой свет на грид.
- Настроение: сдержанная премиальность, «ночной клуб».
- Референсы по ощущению: вечерний клуб, high limit room (без копирования конкретных локаций).

**Вариант 2: Fantasy-Realism**
- Окружение: глубина (небо/облака/даль), магические акценты (свечение, частицы), «живой» фон.
- Настроение: эпичность + читаемость символов и UI.
- Референсы по ощущению: сказочный/мифологический мир с реалистичной подачей (без конкретных IP).

### Цветовая система

| Роль | Hex | Использование |
|------|-----|----------------|
| Primary BG | #0D0D12 | Фон сцены, затемнение |
| Secondary BG | #1A1A24 | Панели, карточки |
| Surface | #252532 | Кнопки idle, поля ввода |
| Accent | #E8B84A | CTA (Spin), важные кнопки, золотые элементы |
| Accent Hover | #F5D06A | Hover на акцентных кнопках |
| Success / Win | #4ADE80 | Выигрыш, положительный баланс |
| Danger / Loss | #F87171 | Ошибки, недостаточно средств |
| Text Primary | #FFFFFF | Заголовки, баланс, ставка |
| Text Secondary | #A1A1AA | Подписи, RTP, правила |
| Neon / Glow | #00D4FF | Подсветка линий, неон на символах (опционально) |

**Правила контраста**: текст на фоне — минимум 4.5:1 (WCAG AA). Акцентные кнопки — контраст к фону не ниже 3:1.

### Типографика

- **Шрифт 1 (заголовки, цифры баланса/выигрыша)**: sans-serif, bold/heavy. Пример: семейство в духе «Impact» по весу, но с нормальной читаемостью (не деформировать).  
  - Mobile: 24–32px для баланса/win, 18–22px для заголовков панелей.  
  - Desktop: 28–40px баланс/win, 22–26px заголовки.
- **Шрифт 2 (тело, правила, paytable)**: sans-serif, regular/medium.  
  - Mobile: 14–16px body, 12px captions.  
  - Desktop: 16–18px body, 14px captions.

Веса: Regular 400, Medium 500, SemiBold 600, Bold 700. Не более двух семейств в одной сцене.

### Материалы UI

- **Стекло**: панели HUD — background с blur (если производительность позволяет) или полупрозрачный градиент (alpha 0.85–0.95), border 1px rgba(255,255,255,0.15).
- **Металл**: кнопки Spin/Auto — градиент (светлый верх, тёмный низ), highlight по верхнему краю 1–2px, тень под кнопкой 4–8px blur.
- **Неон**: только для акцентов (линии выигрыша, активные символы) — outer glow 8–16px, blend mode add/screen.
- **Тени**: UI — drop shadow 4px Y, 8px blur, alpha 0.3; символы в ячейках — лёгкая тень под символом для отрыва от фона.
- **Глубина**: 2–3 слоя (фон сцены → рельсы/рамка грида → символы → оверлеи). Z-order явно задан в спецификации компонентов.

### Символы (реализм и консистентность)

- Единый стиль: либо «реалистичный иллюстративный» (обводка 1–2px тёмная), либо «софт 3D» (градиенты, лёгкий объём). Не смешивать стили в одной игре.
- Размер в ячейке: символ занимает 75–85% высоты ячейки по контенту; отступ от краёв — для читаемости и bounce.
- Глянец: один общий highlight (градиент или overlay) сверху/сбоку, не перегружать.
- Блики: статичный верхний блик + опционально анимация блика при win (проход по символу 200–400 ms).
- Окантовка: тёмная 1–2px по контуру символа для отделения от фона ячейки; при win — цветная (золотая/неон) 2–3px, анимация 300–500 ms.

## B) UI Layout Spec

### Grid и safe area

- **Base canvas**: 1080×1920 (9:16 portrait) — primary; 1920×1080 (16:9) — landscape.
- **Safe area**: отступ от краёв 48px (9:16) и 56px (16:9) для вырезов и жестов. Критичный контент и кнопки — внутри safe area.
- **Grid**: 12 колонок (логических). Отступ между «колонками» контента 24px (mobile), 32px (desktop при 16:9).

### Размеры компонентов (9:16, 1080×1920)

| Элемент | Размер (px) | Позиция (от центра или края) |
|---------|-------------|------------------------------|
| ReelGrid (внешняя рамка) | 900×540 | Центр по X, Y = 0 (по вертикали по макету) |
| Одна ячейка рельса | 172×172 (≈5 колонок по 172 + зазоры) | Внутри грида: 5 колонок × 3 ряда |
| Зазор между символами | 8px | Между ячейками |
| BetPanel (высота) | 160px | Нижняя зона, над safe area |
| Кнопка Spin | 200×80 (min touch 88×88 зона) | В BetPanel, справа |
| Кнопки bet +/- | 72×72 | BetPanel |
| AutoSpin кнопка | 120×56 | Рядом с Spin |
| HUD баланс/ставка | Текст 28px, контейнер ~140×60 | Верх, слева и справа, отступ 48px от края |
| Меню (бургер/настройки) | 56×56 | Верх, угол |
| Звук | 48×48 | Верх, угол |

### Размеры (16:9, 1920×1080)

- ReelGrid: 1000×600, ячейка ~196×196, зазор 10px.
- BetPanel высота: 120px. Spin: 220×88. Остальное масштабируется пропорционально от 9:16 с приоритетом читаемости текста (мин. 14px).

### Состояния кнопок

| Состояние | Визуал | Длительность перехода |
|-----------|--------|------------------------|
| Idle | Базовая текстура, лёгкий gradient | — |
| Hover | Highlight +5% яркости, scale 1.02 | 80 ms ease-out |
| Pressed | Scale 0.96, затемнение −10% | 60 ms ease-in |
| Disabled | Desaturate ~70%, alpha 0.6 | 150 ms |
| Cooldown (после спина) | Без изменений или лёгкий pulse (opacity 0.9↔1) до разблокировки | До получения outcome |

Все интерактивные элементы: минимальная зона нажатия 88×88 px (9:16).

### Micro-interactions

- **Hover**: scale 1.02, яркость +5%, 80 ms ease-out.
- **Press**: scale 0.96, 60 ms ease-in; при отпускании — возврат 100 ms ease-out.
- **Bounce (кнопка Spin после победы)**: опционально лёгкий scale 1 → 1.05 → 1 за 200 ms (ease-out) при появлении кнопки снова.
- **Переключатели (AutoSpin, звук)**: toggle — сдвиг/заливка 120 ms ease-in-out.

## C) Reel Spin Feel (критично)

### Фазы вращения одного барабана

1. **Start (разгон)**: 0–150 ms. Easing: ease-out. Визуально символы уже движутся с начальной скорости (не с нуля — сразу задаём скорость strip).
2. **Steady**: 400–800 ms (зависит от позиции барабана, см. ниже). Линейное движение вниз (или вверх), скорость постоянная. Motion blur включён.
3. **Stop (торможение)**: 180–220 ms. Easing: ease-out-cubic или custom curve (быстрое замедление в конце). Финальная позиция — ровно по outcome (snap to symbol).

### Скорости и задержки по барабанам

- **Скорость strip** (в steady): 2500–3500 px/s (на 1080p). Единое значение для всех барабанов.
- **Задержка старта** (относительно первого барабана): Reel 0: 0 ms; Reel 1: 80 ms; Reel 2: 160 ms; Reel 3: 240 ms; Reel 4: 320 ms. Или: 0, 100, 200, 300, 400 ms для более «каскадного» ощущения.
- **Задержка стопа**: Reel 0: stop at T; Reel 1: T+100 ms; Reel 2: T+200 ms; Reel 3: T+300 ms; Reel 4: T+400 ms. Итого ~100 ms между стопами соседних барабанов (диапазон 80–120 ms).

### Bounce overshoot при стопе

- После остановки по outcome: смещение в сторону движения на 4–8 px, затем возврат (spring или ease-out) за 80–120 ms. Overshoot: 1.02–1.05 от размера ячейки (визуально «дотягивание»).
- Допуск: не более 8 px смещения и 120 ms длительности, чтобы не мешать чтению символов.

### Motion blur

- **Вариант A (шейдер)**: в PixiJS — простой blur по направлению движения (direction 0, −1), strength 2–4 px, только во время steady + начало stop. Отключать за 50 ms до финального snap.
- **Вариант B (без шейдера)**: подмена strip на предрендеренный blurred strip (дубликат символов, размытый по вертикали) на время steady; за 100 ms до стопа — плавный crossfade на sharp strip. Бюджет: 1 доп. текстура на барабан.

### SFX hooks (тайминги относительно анимации)

| Событие | Момент | Допуск (ms) |
|---------|--------|-------------|
| Spin start | Нажатие Spin, до первого движения strip | 0 |
| Reel tick (опционально) | Каждые 80–100 ms во время steady по одному барабану | ±20 |
| Reel stop | Момент snap барабана в финальную позицию | 0 … +30 |
| Win present start | После стопа последнего барабана | +50 … +100 |
| Big Win trigger | После определения win amount, до входа оверлея | 0 … +200 |

## D) Win Presentation (как у топов)

### Подсветка выигрышных символов

- **Включение**: за 50 ms после стопа последнего барабана. Параллельно: glow, scale, (опционально) particles.
- **Glow**: outer glow 12–20 px, цвет золотой/неон, alpha 0.6→0.9 за 150 ms (ease-out). Держать 400–600 ms.
- **Pulse**: scale 1 → 1.08 → 1.03 за 250 ms (ease-out), затем лёгкий hold 1.03 или возврат к 1 за 200 ms.
- **Particles**: 4–8 частиц на символ, вылет от центра символа за 200–400 ms; размер 4–12 px, fade out. Не более 40 частиц суммарно на один win.
- **Блик**: один проход по символу (gradient overlay слева направо) за 200–300 ms, начинается вместе с glow.

Тайминг: glow start 0 ms, scale peak 80 ms, particles 0–300 ms, блик 50–250 ms. Общая длительность «подсветки символов» перед показом win amount: 400–600 ms.

### Paylines / Ways trace

- Линии рисуются поверх грида. Толщина: 4–6 px (mobile), 6–8 px (desktop). Цвет: градиент золотой или неон (#E8B84A → #00D4FF), alpha 0.9.
- **Анимация**: «пробег» по пути линии за 200–350 ms на линию (ease-out). Варианты: stroke dash offset от 1 до 0, или line draw от начала к концу. Одновременно можно 1–2 линии; при 3+ — последовательно с шагом 80–100 ms.
- Исчезновение: fade out 200 ms после показа win counter или по таймеру 1.5–2 s от начала.

### Win counter (count-up)

- Старт: через 400–600 ms после стопа последнего барабана (после подсветки символов).
- Длительность: 800–1200 ms для сумм до ~50× bet; 1200–1800 ms для больших (формула: base 600 ms + 30 ms за каждые 10× bet, cap 2000 ms).
- Easing: ease-out (быстро в начале, замедление в конце). Обновление числа: каждые 16–33 ms (60–30 FPS), значение интерполировано.
- **Anticipation**: перед стартом count-up — пауза 100–150 ms и лёгкий scale контейнера 1 → 1.02 (опционально короткий звук «tick»).

### Big Win sequence

Пороги (в ×Bet): **Big Win** 10×, **Mega Win** 25×, **Ultra Win** 100×. Ниже 10× — только win counter и подсветка, без полноэкранного оверлея.

1. **Entry (0–200 ms)**  
   - Экран: dim до 0.5–0.6 яркости фона за 150 ms.  
   - Burst: 8–12 лучей/частиц из центра грида, 150 ms.  
   - Оверлей (логотип «Big Win» / «Mega Win» / «Ultra Win»): scale 0.3→1.1, ease-out-back, 200 ms.

2. **Build-up (200–800 ms)**  
   - Частицы: конфетти/звёзды из верхней части экрана, плотность средняя.  
   - Усиление: логотип лёгкий pulse (scale 1.0↔1.05), 400 ms.  
   - Win amount появляется под логотипом, count-up стартует (см. выше).

3. **Finale (после count-up)**  
   - Лёгкий camera shake: 2–4 px смещение по X/Y, 2–3 цикла за 200 ms, затем затухание.  
   - Конфетти пик: дополнительный выброс в момент окончания count-up, 300 ms.  
   - Держать экран 600–1000 ms (показать финальную сумму).

4. **Exit (600–1000 ms)**  
   - Fade out оверлея и затемнения за 400 ms.  
   - Возврат к игровому экрану: кнопка Spin снова активна через 200 ms после конца fade.

Общая длительность Big Win: ~3–4 s (без учёта длинного count-up для Ultra).

## E) BonusOverlay / Free Spins

### Интро Free Spins

- Длительность: 3–4 s от появления до кнопки «Start» или автостарта.
- Ключевые кадры:  
  - 0 ms: затемнение, появление контейнера (fade + scale 0.8→1, 300 ms).  
  - 300 ms: заголовок «FREE SPINS» (scale 0→1, ease-out-back, 250 ms).  
  - 600 ms: счётчик «10» или «15» (анимация цифр/карточки, 300 ms).  
  - 1200 ms: опционально правила (1–2 строки, fade in 200 ms).  
  - 2500 ms: кнопка «Start» появляется (fade + bounce 200 ms).  
- Переход в игру: по нажатию Start — fade out оверлея 300 ms, затем первый бонус-спин без дополнительной задержки.

### Retrigger

- Отдельный короткий оверлей: 1.5–2 s.  
- Схема: затемнение 200 ms → текст «RETRIGGER!» + количество доп. спинов (scale 0.5→1.2→1, 400 ms) → hold 800 ms → fade out 300 ms → возврат к барабанам.  
- Без полноценного «интро», только акцент на факте ретриггера.

### Sticky UI в бонусе

- Полоска над или под гридом: «FREE SPINS 7 / 10» (или «7 left»), «MULTIPLIER x2». Высота 48–56 px, полупрозрачный фон, текст 20–24 px. Всегда видима во время бонусных спинов.
- При ретриггере: обновление числа с лёгкой анимацией (scale 1.1→1, 150 ms).

### Skip / отключение анимаций

- **Skip**: кнопка «Skip» появляется через 1–2 s после начала интро Free Spins (или после первого выигрышного спина в бонусе — по дизайну). Нажатие — немедленный переход к спину/следующему действию. Анимация skip не прерывается на середине кадра — переход в следующий логический шаг.
- Правило: при отключении «Animations» в настройках — длительности Big Win и интро сокращаются до минимума (например 500 ms затемнение + 300 ms появление текста), без конфетти и тяжёлых эффектов. Win counter остаётся, но ускоренный (300–500 ms).

## F) Asset Pipeline (Spine / AE / Spritesheets)

### Spine 2D

- **Использовать для**: персонажи (если есть), анимированные рамки грида, глиттер/блеск по символам, сложные win-элементы (руки, предметы). Не для статичных символов рельсов и простых кнопок.
- Экспорт: .skel + .atlas (или .json), один скин на вариант. Костная анимация — до 30 костей на персонажа; анимации до 60 FPS при необходимости, иначе 30 FPS.

### Spritesheets (symbols, UI, VFX)

- **Символы**: один атлас на тему (например symbols_main_001.png + .json). Размер ячейки символа в атласе: 256×256 или 512×512 (для 2x). Padding 2–4 px, trim. Имена: `reel_symbol_01`, `reel_symbol_02`, … `reel_symbol_12`, `reel_symbol_scatter`, `reel_symbol_wild`.
- **UI**: атласы по экранам/зонам. Примеры: `ui_betpanel_001`, `ui_btn_spin_idle`, `ui_btn_spin_pressed`, `ui_btn_autospin`, `ui_hud_balance_bg`. Кнопки — все состояния в одном атласе.
- **VFX**: `vfx_bigwin_burst`, `vfx_particle_star`, `vfx_confetti_*`, `vfx_line_glow`. Отдельные атласы для тяжёлых эффектов (Big Win, Free Spins intro).

### After Effects (PNG sequence)

- Использовать для: интро Free Spins, Big/Mega/Ultra Win, ретриггер, сложные переходы. Экспорт: PNG sequence, 30 FPS (или 24), alpha, premultiplied. Разрешение: 1080×1920 или 1920×1080 по макету.
- Naming: `fs_intro_00000.png` … `fs_intro_00120.png`, `bigwin_entry_00000.png` … Конвертация в спрайты или воспроизведение по кадрам в PixiJS (AnimatedSprite).

### Форматы и настройки

- **PNG**: 8-bit RGBA, premultiplied alpha для корректного blend в WebGL. Сжатие: стандартное (PNG-8 для flat UI где возможно).
- **Размеры атласов**: 2048×2048 основной лимит; при превышении — несколько атласов по зонам (symbols, UI, VFX). Макс 4096 только при необходимости и проверке на слабых устройствах.
- **Scale variants**: 1x (база 1080p), 2x для Retina/высокого DPI. Naming: `atlas_1x.json` / `atlas_2x.json` или суффикс `@2x` в имени текстуры.

### Naming conventions

- Рельсы: `reel_01_symbol_*`, `reel_symbol_*`.
- UI: `ui_btn_spin_*`, `ui_panel_bet_*`, `ui_hud_*`, `ui_modal_*`.
- VFX: `vfx_bigwin_*`, `vfx_fs_intro_*`, `vfx_particle_*`, `vfx_line_*`.
- Звуки: `sfx_spin_start`, `sfx_reel_stop_01` … `sfx_reel_stop_05`, `sfx_win_small`, `sfx_bigwin`, `sfx_fs_trigger`.

## G) Performance & Quality Checklist

### Бюджеты

- **Draw calls**: целевой лимит до 30 на кадр в типичной сцене (грид + HUD + несколько эффектов). Batching символов рельсов в один draw call где возможно.
- **Текстуры**: суммарно в памяти не более 80–100 MB (на 1x); на 2x — осторожно, тестировать на 2 GB RAM устройствах.
- **Частицы**: не более 80–100 активных частиц одновременно (без учёта статичных спрайтов). При Big Win — временный пик до 120, не более 2 s.

### Правила

- No overdraw: не рисовать непрозрачные объекты друг за другом без необходимости; порядок отрисовки — фон → грид → символы → линии → UI → оверлеи.
- Batching: одинаковые текстуры/материалы — в одном batch; символы из одного атласа — один контейнер с общим texture.
- Mipmaps: отключены для UI (чёткость); включены для фоновых текстур большого размера при 3D или масштабировании.
- Динамическое качество: при падении FPS ниже 50 — отключение motion blur и сокращение частиц на 50%; ниже 40 — отключение glow и упрощение шейдеров. Флаг в настройках: «Low / Medium / High / Auto».

### Accessibility

- Читаемость: контраст текста и кнопок по WCAG AA; минимальный размер шрифта 12 px (captions).
- Цвет: не полагаться только на цвет (например «красный = проигрыш»); дублировать иконкой или текстом.
- Звук: переключатель «Sound On/Off» в HUD; критичные события (Big Win, бонус) дублировать визуально.
- Вибро: опциональная вибрация при стопе барабанов и при выигрыше (short pattern), только при включённой настройке и поддержке устройства.

## H) Deliverables

### Список экранов

1. Game (основной экран: грид, HUD, BetPanel, кнопки).
2. Paytable / Rules (модальное окно или отдельный экран).
3. Settings (звук, вибрация, качество, лимиты — если в игре).
4. Free Spins intro (полноэкранный оверлей).
5. Free Spins game (игровой экран с sticky-полоской бонуса).
6. Big Win / Mega Win / Ultra Win (полноэкранные оверлеи).
7. Retrigger (короткий оверлей).
8. AutoSpin settings (модальное окно: количество, стоп-условия).

### Список анимаций

- Reel spin (start, steady, stop) × 5 барабанов.
- Reel stop bounce (per reel).

### Acceptance criteria (визуальное тестирование)

- [ ] Все рельсы останавливаются в позициях, совпадающих с outcome с сервера (pixel-perfect или допуск 1 px).
- [ ] Задержки между стопами барабанов в диапазоне 80–120 ms, визуально последовательно.
- [ ] Win-символы подсвечиваются не позднее 100 ms после последнего стопа.
- [ ] Win counter заканчивается на значении, равном переданному win amount.
- [ ] Big Win оверлей показывается при win ≥ 10× bet; Mega при ≥ 25×; Ultra при ≥ 100×.
- [ ] На 9:16 все кнопки в safe area и не перекрываются вырезами.
- [ ] Контраст текста баланса и ставки не ниже 4.5:1 к фону.
- [ ] При включённом «Low» качестве нет лагов при 60 FPS на референсном устройстве (например iPhone 11 / mid Android).
- [ ] При отключении звука все визуальные события остаются понятными.

## Signature Moments (3 варианта фишек)

1. **Anticipation glow на двух скаттерах**  
   Когда на первых двух остановившихся барабанах уже есть два скаттера, третий ещё крутится: лёгкое пульсирующее свечение под первыми двумя скаттерами и тихий «подъём» звука (riser). Длительность: до стопа третьего барабана. Делает момент «вот-вот бонус» узнаваемым.

2. **Reel slam**  
   Последний барабан (или все пять) при остановке даёт не только bounce, но и короткий «удар»: лёгкое затемнение экрана на 1 кадр (или 16 ms) + микро-shake 1–2 px + один низкочастотный звук. Применять только при выигрышной комбинации. Усиливает ощущение «вес» стопа.

3. **Win ladder (лестница выигрыша)**  
   При нескольких выигрышных линиях суммы показываются не одной строкой, а «ступеньками»: каждая следующая линия добавляет строку под предыдущей с короткой анимацией появления (slide + fade 80 ms) и лёгким увеличением итога. Итоговая сумма внизу с count-up. Создаёт нарастание и читаемость «откуда сколько».
