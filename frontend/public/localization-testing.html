Currency Display Validation FINANCIAL

iGaming platforms must display monetary values correctly across currencies. Tests verify that balance, bet amounts, and win displays render the correct currency code and decimal formatting for each supported currency. 
const currencies = [
  { code: 'USD', symbol: 'USD', balance: 1500.50 },
  { code: 'EUR', symbol: 'EUR', balance: 2300.75 },
  { code: 'GBP', symbol: 'GBP', balance: 980.00 },
  { code: 'UAH', symbol: 'UAH', balance: 45000.00 },
  { code: 'BRL', symbol: 'BRL', balance: 7500.25 },
];

for (const { code, symbol, balance } of currencies) {
  test(`displays ${code} currency code in balance HUD`, async ({ page, context }) => {
    await context.route('**/api/v1/game/init', (route) =>
      route.fulfill({
        status: 200,
        contentType: 'application/json',
        body: JSON.stringify(makeInitResponse(code, balance)),
      })
    );
    const balancePanel = page.getByTestId('hud-balance');
    await expect(balancePanel).toContainText(symbol);
  });
}

Layout Resilience UI

Different locales produce text of varying lengths. German and Ukrainian translations are significantly longer than English. Tests verify that UI elements don't overflow, truncate, or cause horizontal scrollbars when rendering longer strings. 
test('CV landing page renders without horizontal overflow', async ({ page }) => {
  await page.goto('/');
  const hasOverflow = await page.evaluate(() => {
    return document.documentElement.scrollWidth > document.documentElement.clientWidth;
  });
  expect(hasOverflow).toBe(false);
});

Numeric & Date Formatting DATA

Number formatting varies by locale — decimal separators (. vs ,), thousands separators, and date formats. Tests ensure balance displays, bet amounts, and timestamps use consistent formatting. 
test('balance displays decimal separator correctly', async ({ page, context }) => {
  const balancePanel = page.getByTestId('hud-balance');
  const text = await balancePanel.textContent();
  // Balance should contain a decimal number representation
  expect(text).toMatch(/\d+[.,]\d{2}/);
});

String Completeness QA

Missing or untranslated strings break user trust — especially in gambling. Tests scan rendered pages for common i18n antipatterns: empty text nodes, raw translation keys ({{key}}), undefined/null text, and buttons without labels. 
test('no untranslated i18n keys visible on CV page', async ({ page }) => {
  const pageText = await page.textContent('body');
  const i18nKeyPatterns = [
    /\{\{[a-zA-Z_.]+\}\}/,    // {{key.name}}
    /\{t\(['"]/,               // {t('key')}
    /^[A-Z_]{3,}\.[A-Z_]+$/m, // MODULE.KEY_NAME
    /translation_missing/i,
  ];
  for (const pattern of i18nKeyPatterns) {
    expect(pageText).not.toMatch(pattern);
  }
});
