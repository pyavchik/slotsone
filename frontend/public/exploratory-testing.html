Exploratory Testing Approach

Session-based test management (SBTM) adapted for iGaming product testing. Each session operates under a defined charter with a clear mission, time-box, and structured debrief notes to ensure consistent coverage and reproducibility.

Focus areas are derived from a game risk analysis covering four domains: financial integrity, game fairness, UX edge cases, and session management. The risk model prioritizes scenarios where failures would have direct regulatory or monetary impact.

  • Each session has a charter (mission), time-box, and structured debrief notes
  • Focus areas derived from game risk analysis: financial integrity, game fairness, UX edge cases, session management
  • All sessions documented with: setup, observations, bugs found, questions raised
  • Test heuristics applied: CRUD, boundary values, state transitions, concurrency, error handling
  • Findings cross-referenced with automated test results (JMeter RTP suite, API contract tests)

Risk-Based Focus Areas

Financial Integrity
Balance calculations, transaction atomicity, rounding precision
Game Fairness
RTP accuracy, RNG distribution, paytable correctness
Session Management
JWT lifecycle, concurrent sessions, reconnection
User Experience
Responsive layout, animation states, error messaging
