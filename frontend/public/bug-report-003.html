Lobby header displays “PyavchikCasino” instead of the correct product name “SlotsOne”

Environment Any browser · Any device · pyavchik.space/slots · SlotsOne v1.0

Description

The lobby page header at /slots renders the brand name "PyavchikCasino" instead of the correct product name "SlotsOne". This is a branding / white-label misconfiguration — the wrong product identity is displayed to every authenticated user on the main game lobby.

"Pyavchik" is the developer’s surname, not the product brand. The hardcoded strings were likely used as placeholder text during development and never replaced with the correct brand before release.

Preconditions

  • User is authenticated
  • Navigated to the /slots lobby page

Steps to Reproduce

1

Log in to the application

2

Navigate to pyavchik.space/slots

3

Observe the brand name in the top-left header area

It reads "PyavchikCasino" instead of "SlotsOne".

Expected vs Actual Result

✓ Expected

The lobby header displays the correct product brand: “SlotsOne” (or a variant like “SlotsOne”), matching the project name, domain references, and all other branding throughout the application.

✗ Actual

The header displays “PyavchikCasino” — the developer’s surname concatenated with “Casino”. This contradicts the product identity used everywhere else (page titles, API paths, admin panel, documentation).

Source Code

S frontend/src/components/lobby/LobbyHeader.tsx:30-33
// Lines 30-33 of LobbyHeader component <h1 className="lobby-brand"> <span className="lobby-brand-main">Pyavchik</span> <span className="lobby-brand-accent">Casino</span> </h1>
Hardcoded placeholder strings. No configuration, env variable, or brand constant is used.

Evidence

WRONG BRAND NAME
The lobby header at /slots displays “PyavchikCasino”. Every other surface in the application refers to the product as “SlotsOne”.

Root Cause Analysis

The LobbyHeader component has the brand name hardcoded as two literal strings: "Pyavchik" and "Casino". These were used as developer placeholders and were never updated to reflect the actual product name before deployment.

1. Developer creates LobbyHeader component with placeholder brand text
2. Brand strings are hardcoded in JSX — no centralized brand constant or config
3. Placeholder text "Pyavchik" + "Casino" is never replaced with "SlotsOne" WRONG
4. Deployed to production — all users see incorrect brand identity WRONG
PROPOSED FIX

Replace hardcoded brand with correct product name

Update the two <span> elements in LobbyHeader.tsx to display the correct product name.
