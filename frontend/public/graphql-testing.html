Schema Introspection SCHEMA

GraphQL's self-documenting nature enables schema-driven testing. Introspection queries verify that the schema exposes the expected types (Player, Game, GameRound, Wallet, SpinResult), root query fields (me, games, wallet, gameHistory), and mutation fields (login, register, spin, initGame). 
it('schema contains expected types', async () => {
  const { body } = await gqlRequest(`{
    __schema {
      types { name kind }
    }
  }`);

  const typeNames = body.data.__schema.types.map(t => t.name);
  const expected = ['Player', 'Game', 'GameRound', 'Wallet', 'SpinResult'];

  for (const type of expected) {
    assert.ok(typeNames.includes(type),
      `Schema should contain type "${type}"`);
  }
});

Authentication Mutations AUTH

Tests validate the full auth flow via GraphQL mutations: register returns access_token and token_type, login works with valid credentials, and wrong password returns structured GraphQL errors (not HTTP errors). 
it('register mutation returns access token', async () => {
  const { body } = await gqlRequest(
    `mutation Register($input: RegisterInput!) {
      register(input: $input) {
        accessToken
        tokenType
        expiresIn
      }
    }`,
    { input: { email: testEmail, password: 'TestPass123!' } }
  );

  assert.ok(body.data.register.accessToken);
  assert.equal(body.data.register.tokenType, 'bearer');
});

it('wrong password returns structured error', async () => {
  const { body } = await gqlRequest(loginMutation,
    { input: { email: testEmail, password: 'Wrong!' } }
  );
  assert.ok(body.errors);
  assert.ok(body.errors[0].message.toLowerCase().includes('invalid'));
});

Game Session Mutations GAME

Tests exercise the core iGaming flow via GraphQL: initializing a game session (returns config, balance, sessionId) and executing a spin (returns outcome with reel matrix, win/loss, updated balance). 
it('initGame returns session and config', async () => {
  const { body } = await gqlRequest(
    `mutation InitGame($input: InitGameInput!) {
      initGame(input: $input) {
        sessionId
        gameId
        config { reels rows rtp }
        balance { amount currency }
      }
    }`,
    { input: { gameId: 'slot_mega_fortune_001', currency: 'USD' } },
    token
  );

  assert.ok(body.data.initGame.sessionId);
  assert.ok(body.data.initGame.config.reels > 0);
});

it('spin returns outcome and updated balance', async () => {
  const { body } = await gqlRequest(spinMutation, {
    input: { sessionId, gameId, bet: { amount: 0.1, currency: 'USD', lines: 20 } }
  }, token);

  assert.ok(body.data.spin.spinId);
  assert.ok(Array.isArray(body.data.spin.outcome.reelMatrix));
});

Pagination & Filtering QUERY

Game history queries support cursor-based pagination (limit/offset) and field-level filtering (by gameId). Tests verify pagination metadata (total, hasMore) and that filters correctly constrain results. 
it('gameHistory supports limit and offset', async () => {
  const { body } = await gqlRequest(
    `query History($limit: Int, $offset: Int) {
      gameHistory(limit: $limit, offset: $offset) {
        items { id gameId createdAt }
        total
        hasMore
      }
    }`,
    { limit: 5, offset: 0 },
    token
  );

  assert.ok(Array.isArray(body.data.gameHistory.items));
  assert.ok(body.data.gameHistory.items.length <= 5);
  assert.ok(typeof body.data.gameHistory.hasMore === 'boolean');
});

Error Handling ERRORS

GraphQL returns errors differently from REST — always HTTP 200 with an errors array. Tests verify proper error responses for malformed queries, unknown fields, unauthenticated access, and invalid mutation inputs. 
it('returns error for malformed query', async () => {
  const { body } = await gqlRequest('{ this is not valid }');
  assert.ok(body.errors);
});

it('returns error for unauthenticated protected query', async () => {
  const { body } = await gqlRequest(`{ wallet { balance } }`);
  assert.ok(body.errors);
  assert.ok(body.errors[0].message.includes('unauthorized'));
});

Performance PERF

Tests verify that nested queries resolve without N+1 performance degradation. 
it('nested query resolves under 5s', async () => {
  const start = performance.now();
  const { body } = await gqlRequest(`{
    gameHistory(limit: 10) {
      items { id gameId bet { amount } win { amount } }
    }
  }`, undefined, token);
  assert.ok(performance.now() - start < 5000);
});
