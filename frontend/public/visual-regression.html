Screenshot Comparison Strategy VISUAL

Visual regression tests capture baseline screenshots and compare them against future runs. Any pixel difference beyond the configured threshold triggers a failure, catching unintended CSS changes, layout shifts, or broken assets before they reach production. 
const SCREENSHOT_OPTIONS = {
  maxDiffPixelRatio: 0.02,  // Allow 2% pixel difference
  threshold: 0.3,            // Per-pixel color sensitivity
};

test('CV landing page', async ({ page }) => {
  await page.goto('/');
  await expect(page.getByTestId('cv-title')).toBeVisible();
  await page.waitForLoadState('networkidle');

  await expect(page).toHaveScreenshot('desktop-cv-landing.png', {
    fullPage: true,
    ...SCREENSHOT_OPTIONS,
  });
});

Multi-Viewport Coverage DEVICES

Screenshots are captured at three viewport sizes — Desktop (1280x720), iPhone SE (375x667), and iPad Mini (768x1024). This ensures visual consistency across form factors and catches responsive breakpoint issues. 
// Desktop
test.use({ viewport: { width: 1280, height: 720 } });
test('CV landing page', ...);
test('auth login page', ...);
test('game page HUD', ...);

// Mobile
test.use({ ...devices['iPhone SE'] });
test('CV landing page — mobile', ...);
test('auth login page — mobile', ...);

// Tablet
test.use({ ...devices['iPad Mini'] });
test('CV landing page — tablet', ...);

Component-Level Snapshots COMPONENT

Full-page screenshots catch broad changes but can be noisy. Component-level screenshots isolate individual UI sections (action bar, auth form, game HUD) for more precise visual assertions. 
test('CV action bar renders consistently', async ({ page }) => {
  await page.goto('/');
  const actionsBar = page.locator('.cv-actions').first();

  await expect(actionsBar).toHaveScreenshot('component-cv-actions.png', {
    ...SCREENSHOT_OPTIONS,
  });
});

test('auth form component renders consistently', async ({ page }) => {
  await page.goto('/login');
  const form = page.locator('form').first();

  await expect(form).toHaveScreenshot('component-auth-form.png', {
    ...SCREENSHOT_OPTIONS,
  });
});

Theme Testing THEME

Tests verify visual consistency under both dark and light color schemes by using Playwright's emulateMedia() to simulate prefers-color-scheme. This catches theme-related regressions. 
test('prefers dark color scheme', async ({ page }) => {
  await page.emulateMedia({ colorScheme: 'dark' });
  await page.goto('/');
  await expect(page).toHaveScreenshot('theme-dark-cv.png', {
    fullPage: true,
    ...SCREENSHOT_OPTIONS,
  });
});

test('prefers light color scheme fallback', async ({ page }) => {
  await page.emulateMedia({ colorScheme: 'light' });
  await page.goto('/');
  await expect(page).toHaveScreenshot('theme-light-cv.png', { ... });
});

Workflow & CI Integration CI

How visual regression works in CI:
• First run generates baseline screenshots (committed to repo)
• Subsequent runs compare against baselines
• Failures produce diff images highlighting changed pixels
• Update baselines: npx playwright test --update-snapshots 
# Run visual regression tests
npm run test:e2e:visual

# Update baseline screenshots after intentional UI changes
npx playwright test --grep @visual --update-snapshots

# Review diff report
npx playwright show-report
