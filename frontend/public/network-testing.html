Request Header Validation HEADERS

Every API request in an iGaming platform must include proper authentication and idempotency headers. Tests intercept requests via Playwright's route API and validate that Authorization bearer tokens, Idempotency-Key headers, and Content-Type are correctly attached. 
test('game/init sends Authorization bearer token', async ({ page, context }) => {
  let capturedAuthHeader = '';

  await context.route('**/api/v1/game/init', async (route) => {
    capturedAuthHeader = route.request().headers()['authorization'] ?? '';
    await route.fulfill({ status: 200, ... });
  });

  await page.goto('/slots/mega-fortune');
  expect(capturedAuthHeader).toMatch(/^Bearer .+/);
});

test('spin request includes idempotency-key header', async ({ page, context }) => {
  let capturedKey = '';
  await context.route('**/api/v1/spin', async (route) => {
    capturedKey = route.request().headers()['idempotency-key'] ?? '';
    await route.fulfill({ status: 200, ... });
  });
  // ... trigger spin
  expect(capturedKey.length).toBeGreaterThan(8);
});

API Contract Validation CONTRACT

Tests verify that request and response payloads match the expected API contract. Spin requests must include session_id, game_id, and bet object. Responses must contain balance, outcome, and reel matrix. Catching contract violations at the UI layer prevents silent data corruption. 
test('spin request body contains required fields', async ({ page, context }) => {
  let spinBody = {};
  await context.route('**/api/v1/spin', async (route) => {
    spinBody = route.request().postDataJSON();
    await route.fulfill({ ... });
  });

  // trigger spin...
  expect(spinBody).toHaveProperty('session_id');
  expect(spinBody).toHaveProperty('game_id');
  expect(spinBody).toHaveProperty('bet');
  expect(spinBody.bet.amount).toBeGreaterThan(0);
  expect(spinBody.bet.currency).toBe('USD');
});

Error Response Handling ERRORS

iGaming apps must handle server errors gracefully — no raw stack traces, no frozen UI. Tests simulate 500, 401, 429, and timeout scenarios to verify the app degrades gracefully with user-friendly error states. 
test('app handles 500 server error gracefully', async ({ page, context }) => {
  await context.route('**/api/v1/game/init', (route) =>
    route.fulfill({
      status: 500,
      body: JSON.stringify({ error: 'Internal server error' }),
    })
  );
  await page.goto('/slots/mega-fortune');
  const pageContent = await page.textContent('body');
  expect(pageContent).not.toContain('Unhandled');
  expect(pageContent).not.toContain('Cannot read properties');
});

test('app handles 429 rate limiting', async ({ page, context }) => {
  let requestCount = 0;
  await context.route('**/api/v1/game/init', async (route) => {
    requestCount++;
    if (requestCount <= 1) {
      await route.fulfill({ status: 429, headers: { 'Retry-After': '1' }, ... });
    } else {
      await route.fulfill({ status: 200, ... });
    }
  });
  // App should not crash on rate limit
});

Traffic Analysis MONITOR

Tests verify that pages don't make unnecessary API calls. The CV landing page should never call game APIs. The game page should call init exactly once. These tests catch wasteful network requests that impact performance and server load. 
test('CV page does not make unnecessary API calls', async ({ page }) => {
  const apiCalls = [];
  page.on('request', (request) => {
    if (request.url().includes('/api/')) apiCalls.push(request.url());
  });

  await page.goto('/');
  const gameApiCalls = apiCalls.filter(url =>
    url.includes('/game/') || url.includes('/spin')
  );
  expect(gameApiCalls).toHaveLength(0);
});

test('static assets load without 404s', async ({ page }) => {
  const failedResources = [];
  page.on('response', (response) => {
    if (response.status() >= 400 && !response.url().includes('/api/'))
      failedResources.push(response.url());
  });
  await page.goto('/');
  expect(failedResources.length).toBe(0);
});

Slow Network Simulation PERF

Tests verify the app remains functional under degraded network conditions, simulating slow 3G latency and slow external resource loading. Critical for users on mobile data in emerging markets where betting apps are popular. 
test('app remains functional under slow 3G', async ({ page, context }) => {
  await context.route('**/api/v1/game/init', async (route) => {
    await new Promise(r => setTimeout(r, 2000)); // 2s delay
    await route.fulfill({ status: 200, ... });
  });
  await page.goto('/slots/mega-fortune');
  await expect(page.getByRole('button', { name: /spin/i })).toBeVisible({ timeout: 15000 });
});

Test Execution CLI

Network interception tests can be run standalone or as part of the full suite. The --grep @network tag filter runs only network-related specs, while --headed mode enables visual debugging of intercepted requests. 
# Run the full network test suite
npm run test:e2e:network

# Run with tag filter in headed mode
npx playwright test --grep @network --headed
