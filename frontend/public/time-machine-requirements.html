Pyavchik Time Machine — Software Requirements Specification

SlotsOne — Time Rewind Slot Game  |  v1.0  |  Last updated: 2026-03-19

This document specifies the functional and non-functional requirements for the Pyavchik Time Machine slot game, featuring the original Time Rewind Gamble mechanic.

1. Introduction

1.1 Purpose

Define verifiable requirements for the Pyavchik Time Machine slot game so that development, QA, and stakeholders share a single source of truth. Each requirement is traceable to at least one test case. This document demonstrates the QA ability to write formal SRS-style requirements for a custom iGaming mechanic.

1.2 Scope

Pyavchik Time Machine is a 5×3 reel, 15-payline video slot with a unique Time Rewind Gamble feature. After a losing streak, players may be offered the chance to “rewind time” — paying a premium to replay 5 spins with boosted wild frequencies. The game uses a React/PixiJS frontend and an Express/TypeScript backend (RGS) with provably fair mechanics.

1.3 Intended Audience

QA engineers, game designers, developers, regulators evaluating RNG compliance, and hiring managers reviewing this portfolio for iGaming QA competence.

2.2 Game Description

Pyavchik Time Machine is a medium-high volatility video slot themed around time travel. The 5×3 grid features 9 symbols including the Chrono Wild (substitutes all except Vortex Scatter) and the Vortex Scatter (triggers free spins). The game runs on 15 fixed paylines with configurable bet levels from $0.15 to $150.

2.3 Unique Mechanic: Time Rewind Gamble

After 5 consecutive losing spins, the system has a 35% chance of offering the player a Time Rewind — a set of 5 additional spins played at a premium cost with boosted wild symbol frequency. Three tiers are available:

Standard Rewind
2× bet per spin, +15% wild boost, 5 spins
Safe Rewind
1.5× bet per spin, +10% wild boost, 5 spins
Super Rewind
3× bet per spin, +25% wild boost, 5 spins

The total rewind wager is deducted atomically before the 5 spins execute. Each rewind spin uses a fresh RNG seed for regulatory compliance. Players may always skip the offer.

2.4 Constraints

The application uses virtual currency only (no real-money gambling). Wallets and game rounds persist in PostgreSQL. The rewind feature is exclusive to the Time Machine game (game_id: slot_time_rewind_001).

2.5 Assumptions

Users have a modern browser (Chrome, Firefox, Safari, or Edge, latest two major versions) with WebGL support. The server runs on a single instance behind a Caddy reverse proxy with automatic HTTPS.

6. External Interface Requirements

6.1 API Contracts

All endpoints live under /api/v1/. Request and response bodies are validated with Zod schemas. The Time Rewind endpoint (POST /api/v1/game/rewind) is exclusive to game_id: slot_time_rewind_001. The full OpenAPI 3.0 specification is served at /api-docs.

6.2 Database

PostgreSQL 16+ stores wallets, seed pairs, game rounds, and transactions. Rewind spins are flagged with is_rewind: true in the game_rounds table. All monetary values are stored as BIGINT cents with conversion at the API boundary.

6.3 Browser Support

Latest two major versions of Chrome, Firefox, Safari, and Edge. WebGL is required for the PixiJS canvas renderer. The rewind animation requires CSS transitions support.
